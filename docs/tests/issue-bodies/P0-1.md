## 问题
PR #39 在重命名 `.claude/mcp.json` 的 server 时 (`slack-socket` → `chorusgate`)，主动 env 段新增了一行 `MCP_SENDER_ONLY: "1"`。

diff 证据：
```diff
   "mcpServers": {
-    "slack-socket": {
+    "chorusgate": {
       "command": "cmd",
-      "args": ["/c", "slack-socket-mcp"],
+      "args": ["/c", "chorusgate-mcp"],
       "env": {
+        "MCP_SENDER_ONLY": "1",
         "SLACK_BOT_TOKEN": "${SLACK_BOT_TOKEN}",
         "SLACK_APP_TOKEN": "${SLACK_APP_TOKEN}"
       }
     }
   }
```

## 影响
- 直接违反 STORY-9 spec 验收第 6 条「文档不再把 `MCP_SENDER_ONLY` 作为主路径配置」
- `.claude/mcp.json` 正是"主路径配置"，本 PR 主动把 legacy flag 写进去
- 当前 `src/index.ts` 已删 sender-only 分支（即便有 env 也不会激活），但配置与代码收口目标矛盾
- 未来若加回 sender-only 行为，env 自动激活，破坏 Web API-only 边界
- `.claude/mcp.json.example` 已无此字段，三方不一致

## 修法
删 `.claude/mcp.json:7` 那一行（保留 `SLACK_BOT_TOKEN` / `SLACK_APP_TOKEN`）。改后与 `.claude/mcp.json.example` 一致。

## 验收
- `grep -n MCP_SENDER_ONLY .claude/mcp.json` → 0 处
- `diff .claude/mcp.json .claude/mcp.json.example` → 仅敏感字段/注释差异
