## 问题
`.claude/mcp.json` 实际配置用 Windows-only 的 `cmd /c` 封装：
```json
"command": "cmd",
"args": ["/c", "chorusgate-mcp"]
```

但 `docs/feature-mcp-server.md` 文档写的是跨平台：
```json
"command": "chorusgate-mcp",
"args": []
```

`.claude/mcp.json.example` 跟实际一样用 Windows wrapper。

## 影响
- Mac/Linux 开发者照文档抄配置 → 启动失败（`cmd` 在 Unix 不存在）
- 文档与实际不一致，新用户被骗
- STORY-9 spec 验收第 3 条「`.claude/mcp.json` 可同时被本地 runtime 与 gateway spawn 的 runtime 复用」隐含跨平台可用
- 三方不一致（实际 / example / doc 三个不同的命令格式）

## 修法（二选一）
- **(A) 推荐：** 把 `.claude/mcp.json` 改成跨平台 (`"command": "chorusgate-mcp"`)，与文档一致。`bin/chorusgate-mcp.mjs` 已有 shebang，Unix 可直执。
- **(B)** 若 Windows-only 是项目硬约束，更新 `docs/feature-mcp-server.md` 写明 Windows wrapper 并加跨平台 TODO 注释。

## 验收
- 方案 A：`cat .claude/mcp.json` 与 `docs/feature-mcp-server.md` 文档示例逐字段比对，一致；非 Windows 机器 dry-run `chorusgate-mcp --help`
- 方案 B：文档明确写 Windows wrapper
